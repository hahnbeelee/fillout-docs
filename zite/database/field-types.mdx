Zite DB supports a wide variety of field types to handle different data structures and validation requirements. Each field type has specific configuration options defined in the template property.

## Field Types Overview

Each field type consists of:

1. **Template Structure** - Configuration options for the field (appearance, validation, etc.)
2. **Record Data Structure** - How data is stored when adding/updating records for this field

Fields can be created using the Public API with their input template schemas, which have optional properties with sensible defaults.

---

## Text Fields

### Single Line Text

Use cases: Names, titles, short descriptions
Template: No configuration required

```json
{
  "name": "Full Name",
  "type": "single_line_text",
  "template": {}
}
```

**Record Data Structure:**
```typescript
string; // e.g., "John Doe", "Product Name"
```

---

### Long Text

Use cases: Comments, descriptions, notes
Template: No configuration required

```json
{
  "name": "Description", 
  "type": "long_text",
  "template": {}
}
```

**Record Data Structure:**
```typescript
string; // Multi-line text, e.g., "This is a long description\nwith multiple lines"
```

---

### Email

Use cases: Email addresses with validation
Template: No configuration required

```json
{
  "name": "Contact Email",
  "type": "email", 
  "template": {}
}
```

**Record Data Structure:**
```typescript
string; // Valid email address, e.g., "user@example.com"
```

**Record Data Structure:**

## Number Fields

### Number

```json
{
  "name": "Quantity",
  "type": "number",
  "template": {
    "decimalPlaces": 2,
    "numberFormat": "local"
  }
}
```

**Template Structure (Input):**
- `decimalPlaces` (number, optional): 0-10, default: 1
- `numberFormat` (string, optional): "local" | "comma_period" | "period_comma" | "space_comma" | "space_period", default: "local"

**Record Data Structure:**
```typescript
number | null; // e.g., 42.5, 1000, null
```

---

### Currency

```json
{
  "name": "Price",
  "type": "currency", 
  "template": {
    "currencySymbol": "$",
    "decimalPlaces": 2,
    "numberFormat": "comma_period"
  }
}
```

**Template Structure (Input):**
- `currencySymbol` (string, optional): default: "$"
- `decimalPlaces` (number, optional): 0-10, default: 2
- `numberFormat` (string, optional): "local" | "comma_period" | "period_comma" | "space_comma" | "space_period", default: "local"

**Record Data Structure:**
```typescript
number | null; // Stored as raw number, e.g., 99.99, 1000.00, null
```

---

### Percent

```json
{
  "name": "Completion Rate",
  "type": "percent",
  "template": {
    "decimalPlaces": 1,
    "showProgressBar": true,
    "allowNegative": false
  }
}
```

**Template Structure (Input):**
- `decimalPlaces` (number, optional): 0+, default: 0
- `numberFormat` (string, optional): default: "local"
- `showProgressBar` (boolean, optional): default: false
- `allowNegative` (boolean, optional): default: false

**Record Data Structure:**
```typescript
number | null; // Stored as decimal (0.5 = 50%), e.g., 0.85, 1.0, null
```

---

### Rating

```json
{
  "name": "Customer Rating",
  "type": "rating",
  "template": {
    "maxRating": 5
  }
}
```

**Template Structure (Input):**
- `maxRating` (number, optional): 1-10, default: 5

**Record Data Structure:**
```typescript
number | null; // Integer from 0 to maxRating, e.g., 4, 5, null
```

---

### Duration

```json
{
  "name": "Meeting Length",
  "type": "duration",
  "template": {
    "format": "h:mm:ss"
  }
}
```

**Template Structure (Input):**
- `format` (string, optional): "h:mm" | "h:mm:ss" | "h:mm:ss.s" | "h:mm:ss.ss" | "h:mm:ss.sss", default: "h:mm"

**Record Data Structure:**
```typescript
number | null; // Stored as total seconds, e.g., 3600 (1 hour), 7260 (2:01:00), null
```

## Selection Fields

### Single Select

```json
{
  "name": "Status",
  "type": "single_select",
  "template": {
    "options": [
      {
        "value": "active",
        "label": "Active", 
        "color": "#10b981"
      },
      {
        "value": "pending",
        "label": "Pending",
        "color": "#f59e0b"
      },
      {
        "value": "inactive", 
        "label": "Inactive",
        "color": "#ef4444"
      }
    ],
    "disableColors": false
  }
}
```

**Template Structure (Input):**
- `options` (array, optional): Array of option objects, default: []
  - `value` (string, required): Option value (min length: 1)
  - `label` (string, required): Display label (min length: 1)
  - `color` (string, optional): Color for the option
- `disableColors` (boolean, optional): default: false

**Record Data Structure:**
```typescript
string | null; // Option value, e.g., "active", "pending", null
```

---

### Multiple Select

```json
{
  "name": "Tags",
  "type": "multiple_select",
  "template": {
    "options": [
      {"value": "urgent", "label": "Urgent", "color": "#ef4444"},
      {"value": "feature", "label": "Feature", "color": "#3b82f6"},
      {"value": "bug", "label": "Bug", "color": "#f59e0b"}
    ],
    "disableColors": false
  }
}
```

**Template Structure (Input):**
Same as single_select

**Record Data Structure:**
```typescript
string[]; // Array of option values, e.g., ["urgent", "bug"], []
```

---

### Checkbox

```json
{
  "name": "Is Complete",
  "type": "checkbox",
  "template": {
    "color": "#10b981"
  }
}
```

**Template Structure (Input):**
- `color` (string, optional): default: "#a8a4ac"

**Record Data Structure:**
```typescript
boolean; // true or false
```

## Date & Time Fields

### Date

```json
{
  "name": "Due Date",
  "type": "date",
  "template": {
    "dateFormat": "us"
  }
}
```

**Template Structure (Input):**
- `dateFormat` (string, optional): "local" | "long" | "us" | "european" | "iso", default: "local"

**Record Data Structure:**
```typescript
string | null; // ISO date string (UTC), e.g., "2024-03-15T00:00:00.000Z", null
```

---

### DateTime

```json
{
  "name": "Meeting Time",
  "type": "datetime",
  "template": {
    "displayTimeZone": true,
    "dateFormat": "us",
    "timeFormat": "24h",
    "timezone": "America/New_York"
  }
}
```

**Template Structure (Input):**
- `displayTimeZone` (boolean, optional): default: false
- `dateFormat` (string, optional): "local" | "long" | "us" | "european" | "iso", default: "local"
- `timeFormat` (string, optional): "12h" | "24h", default: "12h"
- `timezone` (string, optional): default: "" (browser timezone)

**Record Data Structure:**
```typescript
string | null; // ISO datetime string, e.g., "2024-03-15T14:30:00.000Z", null
```

## Relationship Fields

### Linked Record

```json
{
  "name": "Related Tasks",
  "type": "linked_record",
  "template": {
    "tableId": "tbl789",
    "allowMultiple": true
  }
}
```

**Template Structure (Input):**
- `tableId` (string, required): ID of table to link to
- `allowMultiple` (boolean, optional): default: true
- `inverseFieldId` (string, optional): ID of inverse field
- `isInverse` (boolean, optional): flag for inverse fields

**Record Data Structure:**
```typescript
string[] | string | null
// - Array of record IDs if allowMultiple: true, e.g., ["rec123", "rec456"]
// - Single record ID if allowMultiple: false, e.g., "rec123"
// - null if no links
```

<Note>
  - `tableId` can be either a table ID or table name during base creation
  - Creating linked record fields automatically connects tables in both directions
</Note>

---

### Lookup

```json
{
  "name": "Task Names",
  "type": "lookup",
  "template": {
    "linkedRecordFieldId": "fld123",
    "lookupFieldId": "fld456"
  }
}
```

**Template Structure (Input):**
- `linkedRecordFieldId` (string, required): ID of linked record field in same table
- `lookupFieldId` (string, required): ID of field to lookup in linked table

**Record Data Structure:**
```typescript
any | any[] | null
// Returns the actual values from the looked-up field
// - Single value if linked record field has allowMultiple: false
// - Array of values if linked record field has allowMultiple: true
// - Type depends on the field being looked up (string, number, boolean, etc.)
```

Lookup fields automatically display values from linked records and update when the source data changes.

<Warning>
  Lookup fields cannot be created during initial base setup. Add them later using the field creation endpoint.
</Warning>

## Primary Field Requirements

The first field in each table becomes the **primary field** and must be one of these types:

- `single_line_text`
- `long_text`
- `date`
- `phone_number`
- `email`
- `url`
- `number`
- `currency`
- `percent`
- `duration`

---

## General Notes

1. **Empty Values**: Each field type has a default empty value:
   - Text fields: `""` (empty string)
   - Number fields: `null`
   - Boolean fields: `false`
   - Array fields: `[]` (empty array)

2. **Required Fields**: When creating fields via API, only `type` and `name` are required. Template properties are optional and will use sensible defaults.

3. **Data Validation**: All record data is validated according to the field type and template configuration when storing/updating records.

4. **Field IDs**: Field IDs are auto-generated if not provided during creation. For API consistency, you can optionally provide your own IDs.

<Warning>
  Changing field types after data has been added may result in data loss if the new type is incompatible with existing values.
</Warning>
